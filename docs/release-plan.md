# 发布与回滚方案（W8-03）

## 1. 发布目标
- 发布范围：规则管理、任务计算、布局查询、模板导出
- 发布窗口：建议工作日低峰（北京时间 20:00 后）
- 回滚目标：接口可在 10 分钟内恢复到上一个稳定版本

## 2. 发布前检查清单
1. 拉取目标 commit 并打 tag（例：`release-2026-03-25`）。
2. 执行数据库备份：`output/packing.db`。
3. 执行全量测试：`python -m unittest discover -s tests`。
4. 验证关键 API：
- `POST /api/plans`
- `POST /api/plans/{id}/calculate`
- `GET /api/layout/{id}`
- `POST /api/plans/{id}/export`

## 3. 发布步骤
1. 停止旧进程（保留日志）。
2. 部署新代码并执行应用启动：
- `python backend_server.py`
3. 进行健康检查：
- `GET /healthz` 返回 `{"status":"ok"}`
4. 执行冒烟：
- 新建计划 -> 计算 -> 确认 -> 导出

## 4. 回滚步骤
1. 停止当前进程。
2. 切回上一稳定 tag/commit。
3. 恢复数据库备份文件 `packing.db`。
4. 重启服务并执行健康检查与冒烟。

## 5. 数据安全策略
- 发布前必须备份数据库文件并保留至少 2 个历史版本。
- 回滚时优先恢复代码，其次恢复数据库，避免 schema 与代码不一致。
- 所有人工操作通过 `audit_log` 保留追溯链路。

## 6. 风险与处置
- 风险：导出模板文件缺失导致导出失败。
- 处置：接口返回明确错误码，允许传入 `template_path` 指定模板。

- 风险：异步计算中断导致计划状态停留在 `CALCULATING`。
- 处置：允许运维触发重算接口，失败会落审计日志。

# 装箱装托系统开发计划（docs/dev-plan）

## 1. 计划范围与依据
1. 依据文档：`soft.md`（V3.0）与 `skill.md`。
2. 依据现状：当前 `demo` 已具备页面骨架与假数据流程，但缺少完整生产级后端、规则引擎与任务闭环。
3. 目标：8 周内完成“导入 -> 计算 -> 方案对比 -> 确认 -> 导出 -> 三维联动”的可上线版本。

## 2. 阶段总览（8周）
| 阶段 | 周期 | 目标 | 核心交付 |
| --- | --- | --- | --- |
| A | 第1周 | 需求冻结与技术方案 | 字段字典、规则映射、接口草案、异常码 |
| B | 第2周 | 数据与规则底座 | 数据库模型、规则导入器、规则快照 |
| C | 第3周 | 装箱引擎 MVP | 三层优先级装箱、装箱约束校验 |
| D | 第4周 | 装托引擎 MVP | 托盘放置、限重/尺寸校验、拼托策略 |
| E | 第5周 | 任务中心后端 | 任务状态机、异步计算、方案存储 |
| F | 第6周 | 前后端联调 | 列表/详情/规则/汇总页面接口化 |
| G | 第7周 | 导出+3D联动 | 模板导出达标、3D数据接口与筛选 |
| H | 第8周 | 测试与发布准备 | 回归测试、压测、发布/回滚清单 |

## 3. 模块拆解与责任面
1. 规则中心：规则导入、冲突检测、版本快照、追溯。
2. 计算引擎：装箱求解、装托求解、候选方案生成、评分。
3. 任务中心：创建任务、触发计算、确认方案、覆盖上传、状态流转。
4. 导入导出：导入校验、模板导出、异常报告。
5. 前端应用：任务流页面、方案对比、3D展示、下载闭环。
6. 可观测性：日志、指标、错误追踪、审计。

## 5. 里程碑与门禁
1. M1（第2周末）：规则可解析、可入库、可追溯。
2. M2（第4周末）：装箱装托引擎均可独立产出可执行方案。
3. M3（第6周末）：前后端业务闭环打通。
4. M4（第7周末）：导出样式达标、3D联动可用。
5. M5（第8周末）：通过回归+压测，进入上线窗口。

## 6. 验收标准（DoD）
1. 规则实现完全覆盖 `skill.md` 的装箱与装托约束。
2. 导出结果与 `导出模板.xlsx` 视觉及字段语义一致。
3. 异常场景可定位（缺规则、超限重、无可用规格）。
4. 关键操作可审计（规则版本、输入快照、最终方案）。
5. 500行订单场景达到性能目标。

## 7. 风险与应对
1. 规则数据缺失或冲突。
- 应对：导入阻断+冲突报告+人工修复台账。
2. 模板结构变化影响导出。
- 应对：模板版本化+映射自动化测试。
3. 算法耗时超标。
- 应对：剪枝、缓存、分层求解与异步化。

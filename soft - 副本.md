# 装箱装托规划系统 - 详细产品文档（V2.0）

## 1. 文档目标
本文件用于统一业务、产品、设计、开发、测试的实现口径，覆盖：
1. 业务流程与规则。
2. 模块与字段定义。
3. 计算引擎方案。
4. 接口与页面行为。
5. 验收标准与实施计划。

## 2. 产品定位与边界
### 2.1 产品定位
面向外贸/供应链发货环节，自动生成“内盒 -> 外箱 -> 托盘”装载方案，减少外箱数与托盘数，同时保障操作复杂度可控与结果可执行。

### 2.2 本期范围（V2.0）
1. 客户模块（基础装箱偏好配置）。
2. 客户装托/装箱单模块（方案生成与最终确认）。
3. 型号-内盒模块（默认映射管理）。
4. 内盒-外箱模块（箱规映射管理）。
5. 装箱装托计算引擎（3 套候选方案）。
6. 三维可视化（内盒/外箱/托盘三级）。
7. 固定模板 Excel 导入/导出。

### 2.3 非本期范围
1. ERP/WMS 实时双向集成。
2. 多模板导出（按客户/国家差异化模板）。
3. 自动机器学习调参。

## 3. 核心业务决策（已确认）
1. 优化优先级固定：托盘数 > 外箱数 > 复杂度。
2. 支持权重配置（用于人工调参），不改变硬约束。
3. A1 固定文案：`合并` / `不合并`。
4. 候选方案固定输出 3 套：保守/均衡/极致省箱。
5. 相近规格判定方式：尺寸差阈值。
6. 最后一托加大加高触发条件：在规定范围内且至少减少 1 托。
7. 允许手工锁定型号独立装箱/独立装托。
8. 三维展示到内盒层级。
9. 导出模板为单一固定样式。

## 4. 端到端业务流程
1. 维护基础数据：客户配置、型号-内盒、内盒-外箱、外箱-托盘。
2. 导入发货计划（订单 Excel）。
3. 选择客户并读取客户默认装箱偏好。
4. 计算引擎生成 3 套候选方案。
5. 用户对比方案并选定最终方案。
6. 如有线下调整，可上传最终 Excel 覆盖系统结果。
7. 输出装箱单/装托单与三维效果链接。
8. 单据归档，可回溯历史版本与规则快照。

## 5. 模块设计
## 5.1 客户模块（基础装箱信息）
### 5.1.1 功能
1. 客户档案新增/编辑/停用。
2. 客户默认装箱要求配置（合并/不合并）。
3. 客户内盒类型与提供方配置。
4. 客户特定内盒替换规则维护（如 104->111）。

### 5.1.2 字段定义
1. 客户编号（唯一）。
2. 客户名称。
3. 客户地址。
4. 装箱要求默认值：`合并`/`不合并`。
5. 内盒类型：枚举（我司彩盒、我司白盒、客户彩盒、自定义）。
6. 内盒提供方：`我司`/`客户`。
7. 特定内盒要求：字符串规则或结构化映射（建议结构化存储）。
8. 状态：启用/停用。
9. 生效时间范围（可选，用于过渡配置）。

### 5.1.3 规则
1. 客户配置优先于订单 A1；但若订单 A1 明确填写，则以订单为准（建议此优先级，便于单次覆盖）。
2. 特定内盒替换在“型号-内盒映射”之后执行。
3. 替换规则支持多条（如 104->111, 105->111）。

## 5.2 客户装托/装箱单模块（计划中心）
### 5.2.1 功能
1. 创建装箱装托任务。
2. 关联发货计划（可多订单）。
3. 保存系统生成的 3 套候选方案。
4. 选择最终方案并冻结。
5. 上传最终 Excel 手工覆盖。
6. 查看装箱三维与装托三维效果。
7. 输出审计日志（谁在何时选择了哪个方案）。

### 5.2.2 字段定义
1. 任务单号（唯一）。
2. 客户编号。
3. 发货计划集合（订单号列表）。
4. 发货时间。
5. 装箱要求（从客户默认带入，可人工修改）。
6. 候选方案列表（3 套，含指标）。
7. 最终方案 ID。
8. 最终方案来源：`系统选择`/`上传覆盖`。
9. 上传覆盖文件地址（可空）。
10. 装箱三维链接。
11. 装托三维链接。
12. 状态：草稿/计算中/待确认/已确认/已归档。

### 5.2.3 状态流转
1. 草稿 -> 计算中 -> 待确认 -> 已确认 -> 已归档。
2. 已确认允许“重新打开”到待确认（保留历史版本）。
3. 上传覆盖仅允许在“待确认”状态执行。

## 5.3 型号-内盒模块
### 5.3.1 功能
1. 维护型号与默认内盒映射。
2. 支持批量 Excel 导入。
3. 支持多 sheet 解析与冲突校验。

### 5.3.2 字段定义
1. 型号。
2. 单只重量（kg）。
3. 默认内盒（支持 `104*2` 语法）。
4. 默认每箱数量。
5. 默认净重/毛重。
6. 默认体积。
7. 默认外箱尺寸规则。
8. 数据来源与版本。

### 5.3.3 规则
1. 缺失型号映射时计算失败并出具异常报告。
2. 同型号重复规则按“版本 + 生效时间”择优。

## 5.4 内盒-外箱模块
### 5.4.1 功能
1. 维护内盒尺寸与外箱映射。
2. 维护外箱-托盘默认关系（含排列方式）。
3. 支持相近规格引用。

### 5.4.2 字段定义
1. 内盒编号。
2. 内盒长宽高（mm）。
3. 外箱规格（cm）。
4. 内盒+外箱重量（kg）。
5. 一箱总数（只）。
6. 内盒排列方式（横*竖*层）。
7. 默认托盘规格（cm）。
8. 默认每托外箱数。
9. 外箱排列方式（横*竖*层）。
10. 相近规格引用组。

### 5.4.3 规则
1. 默认同规格同托，之后才允许相近规格拼托。
2. 相近规格判定使用尺寸差阈值参数。
3. 104/105/111 优先作为拼托补位规格。

## 6. 计算引擎详细方案
## 6.1 输入
1. 订单数据（型号、数量、订单号、单位）。
2. 客户配置。
3. 型号-内盒规则。
4. 内盒-外箱-托盘规则。
5. 算法参数（权重、阈值、锁定规则）。

## 6.2 装箱求解
1. 阶段 A：构建可行解。
2. 遍历优先级：同型号 -> 同内盒 -> 可升级内盒。
3. 执行客户内盒替换规则（如 104->111）。
4. 应用锁定规则（独立装箱型号不参与拼箱）。
5. 阶段 B：局部优化（合并、交换、拆分）以减少箱数和复杂度。

## 6.3 装托求解
1. 按托盘可用尺寸与限重进行放置。
2. 先正放，再处理可竖放补位箱。
3. 相同规格优先同托，相近规格按阈值拼托。
4. 最后一托加大加高策略：仅在允许尺寸内且至少减少 1 托时触发。
5. 应用独立装托锁定规则。

## 6.4 候选方案生成
1. 保守方案：限制拼箱拼托，易执行。
2. 均衡方案：综合箱数与复杂度。
3. 极致省箱方案：优先减少箱/托，允许更高复杂度。

## 6.5 输出指标
1. 外箱总数。
2. 托盘总数。
3. 总毛重与每托毛重。
4. 拼箱次数、拼托次数。
5. 空间利用率（箱内、托盘内）。

## 7. 三维可视化要求
1. 展示层级：内盒、外箱、托盘。
2. 支持旋转、缩放、平移、分层隐藏。
3. 支持按订单号/托盘号筛选。
4. 显示关键标签：规格、数量、重量、方向。
5. 提供截图导出（PNG）与链接分享。

## 8. 导入导出规范
## 8.1 导入
1. 订单 Excel 模板校验（必填列、数据类型、重复校验）。
2. A1 仅接受 `合并` / `不合并`，其他值报错。
3. 错误提示格式：行号 + 字段 + 原因 + 修复建议。

## 8.2 导出
1. 使用单一固定模板。
2. 包含：装箱明细、装托明细、汇总统计、异常信息。
3. 拼箱时支持外箱规格列合并单元格。
4. 导出文件名建议：`客户_订单组合_日期.xlsx`。

## 9. 页面与交互
## 9.1 页面清单
1. 客户管理页。
2. 型号-内盒规则页。
3. 内盒-外箱规则页。
4. 任务列表页。
5. 任务详情页（方案对比 + 三维 + 导出）。

## 9.2 关键交互
1. 新建任务时自动带入客户默认配置。
2. 支持在任务中覆盖默认配置（仅本任务生效）。
3. 方案对比支持一键选择最终方案。
4. 上传覆盖 Excel 后，页面标识“已覆盖”。

## 10. API 设计（建议）
1. `POST /api/customers`：创建客户。
2. `GET /api/customers/{id}`：查询客户。
3. `POST /api/rules/model-inner-box/import`：导入型号-内盒规则。
4. `POST /api/rules/inner-box-outer-box/import`：导入内盒-外箱规则。
5. `POST /api/plans`：创建任务。
6. `POST /api/plans/{id}/calculate`：触发计算。
7. `GET /api/plans/{id}`：查询任务与方案。
8. `POST /api/plans/{id}/finalize`：确定最终方案。
9. `POST /api/plans/{id}/override-excel`：上传覆盖文件。
10. `GET /api/plans/{id}/export`：导出最终 Excel。
11. `GET /api/plans/{id}/layout/{solution_id}`：获取三维布局 JSON。

## 11. 数据库模型（建议）
1. `customer`：客户主表。
2. `customer_box_policy`：客户装箱偏好与内盒策略。
3. `shipment_plan`：装箱装托任务主表。
4. `shipment_plan_order`：任务与订单关系。
5. `solution`：候选方案表（3 条/任务）。
6. `solution_item_box`：方案装箱明细。
7. `solution_item_pallet`：方案装托明细。
8. `rule_model_inner_box`：型号-内盒规则。
9. `rule_inner_outer_pallet`：内盒-外箱-托盘规则。
10. `audit_log`：操作审计日志。

## 12. 权限与审计
1. 计划员：创建任务、计算、导出。
2. 复核员：确认最终方案、上传覆盖。
3. 管理员：维护客户与规则。
4. 全链路审计：配置修改、计算触发、方案确认、覆盖上传。

## 13. 非功能要求
1. 性能：中等规模任务（<= 500 行订单）60 秒内完成。
2. 稳定性：计算任务成功率 >= 99%。
3. 可追溯：输入快照、规则版本、算法参数、结果版本完整留存。
4. 可维护：规则配置化，不硬编码客户逻辑。

## 14. 验收标准
1. 客户模块可独立维护并被任务正确继承。
2. 任务可稳定生成 3 套方案并可对比指标。
3. 最终方案可系统确认或上传覆盖确认。
4. 三维页面可准确显示内盒、外箱、托盘关系。
5. 导出固定模板格式与业务样例一致。
6. 日志可追踪每次关键操作。

## 15. 实施计划（建议）
1. 第 1 阶段：数据模型与规则导入（客户、型号-内盒、内盒-外箱）。
2. 第 2 阶段：装箱装托求解与 3 套方案生成。
3. 第 3 阶段：任务中心、方案确认、覆盖上传、导出。
4. 第 4 阶段：三维展示与性能优化。

## 16. 默认参数建议
1. `objective_weights`: `{ pallet_count: 1.0, box_count: 0.6, complexity: 0.3 }`
2. `candidate_solution_count`: `3`
3. `similar_size_threshold_cm`: `2`
4. `max_pallet_weight_kg`: `1250`
5. `pallet_self_weight_kg`: `30`
6. `last_pallet_expand_max_cm`: `116*116*116`

## 17. 开发前检查清单
1. 确认固定导出模板最终样式。
2. 确认客户内盒类型枚举完整值。
3. 确认订单 Excel 中单位字段与客户编号映射关系。
4. 确认是否需要“部分确认”（只确认部分订单）。
